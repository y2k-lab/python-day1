---

# QA観点（テスト観点）

正常系
- `left → wait → yellow` を入力すると勝利メッセージが表示される

異常系
- 方向入力にスペースのみを入力した場合、Game Over になる
- 色選択で `red / yellow / blue` 以外を入力した場合、Game Over になる
- `wait / swim` 以外を入力した場合、Game Over になる

表示・仕様確認
- 大文字入力（LEFT / WAIT / YELLOW）でも正しく判定される
- 選択肢（wait / swim / red / yellow / blue）が画面上で明確に表示される
- 文言にタイプミスがないこと



#補足
- テストしやすくするため、表示と入力を分離して実装
- ロジックはシンプルさを優先


[### Day５] 

Udemy「100 Days of Code」Day5 の学習記録。

## 内容
ランダムなパスワードを生成するプログラムを作成した。

- 英字（a-z）
- 数字（0-9）
- 記号（! # $ % & ( ) * +）

それぞれの個数を入力し、シャッフルしたパスワードを生成する。

## 使用技術
- Python
- random モジュール

## 工夫点（最小改造）
- 数字以外が入力された場合に `ValueError` を捕捉するため、`try / except` を使用
- 入力値をそのまま使わず、エラーが起きるケースを確認しやすくした

## QA視点（観点）
### 観点1：すべて0を入力した場合
- 入力：letter=0, number=0, symbol=0  
- 期待結果：`Your password is:` のみ表示される

### 観点2：スペースのみを入力した場合
- 入力：`" "`  
- 期待結果：ValueError が発生し、エラーメッセージが表示される

### 観点3：数字以外を入力した場合
- 入力：`abc`  
- 期待結果：ValueError が表示される

### 観点4：正常値入力
- 入力：letter=5, symbol=3, number=1  
- 期待結果：英字5個・記号3個・数字1個のランダムなパスワードが生成される
